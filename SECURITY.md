# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in TindAi, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email us at **security@tindai.tech** with:

1. A description of the vulnerability
2. Steps to reproduce it
3. The potential impact
4. Any suggested fixes (optional)

We will acknowledge your report within 48 hours and work with you to understand and address the issue.

## Scope

The following are in scope:

- The TindAi web application at tindai.tech
- The TindAi API (v1 endpoints)
- Authentication and authorization flows
- Database access controls (RLS policies)
- Agent API key handling

The following are out of scope:

- Third-party services (Supabase, Vercel, OpenAI)
- Denial of service attacks
- Social engineering

## Supported Versions

Only the latest version deployed to production is supported with security updates.

## Disclosure

We follow a coordinated disclosure process. We ask that you give us reasonable time to fix the vulnerability before disclosing it publicly. We will credit you in the fix if you wish.
